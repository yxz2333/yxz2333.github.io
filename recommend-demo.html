增量更新机制

skip

code_hash 命中已有集合
→ 内容没变
→ 跳过 GPU 编码

new

hash 不在集合 + key 不冲突
→ 全新代码块
→ UnixCoder 编码入库

modify

key 相同但 hash 不同
→ 内容被修改
→ 删旧行 + 重编码

delete

旧 hash 没再出现
→ 笔记/代码被删
→ 从向量库移除

SHA256 12位指纹O(1) 查重--force 全量重建

Token TF-IDF 详解

分词 + 过滤

原始代码: priority_queue<pii> pq; int n; cin >> n; for(int i=0; i<n; i++) ... 分词后: priority_queue, pii, pq, n, i (int/for/cin 被关键字过滤)

IDF 效果

priority_queue → 仅 5 篇有 → IDF 高
lca → 仅 3 篇有 → IDF 极高
int → 全部 200 篇有 → IDF ≈ 0
for → 全部 200 篇有 → IDF ≈ 0

稀有算法词自动获得高权重；常见关键字自动归零
